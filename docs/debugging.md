# Debugging

## In the Cloud

### Access Live Environment
1. Visit [Google's cloud console](https://console.cloud.google.com/).
   - Alternatively to these 5 steps [you may setup gcloud locally](#setup-gcloud-locally).
2. Set your project to `restful-webserver`.
3. Activate Cloud Shell (upper-right button or press `g` > `s`).
4. Make sure you see the following prompt:
    ```sh
    USER@cloudshell:~ (restful-webserver)$
    ```
5. Then run the following commands:
    ```sh
    gcloud compute instances list
    ```
    You should see:
    ```yaml
    NAME: web-server
    ZONE: us-west1-a
    MACHINE_TYPE: e2-micro
    PREEMPTIBLE:
    INTERNAL_IP: # removed for safety
    EXTERNAL_IP: # removed for safety
    STATUS: RUNNING
    ```
    Then run:
    ```sh
    # gcloud compute ssh [NAME] --zone [ZONE]
    gcloud compute ssh web-server --zone us-west1-a
    ```

Once inside, you can do several things to debug your server.

- Check if the server is running:
  ```sh
  docker ps
  ```
  One of the entries should list our project:
  ```
  gcr.io/restful-webserver/restful-webserver:latest
  ```
  If the command fails, run:
  ```sh
  ps -e | grep docker
  ```
  to see if Docker is running. If it isn't, something is very wrong.
- To SSH into the server (SSH inception):
  You may use the `CONTAINER ID` or any of its `NAMES` listed in `docker ps` and then:
  ```sh
  docker exec -it [ID/NAME] bash
  ```
  Now you can inspect the files within the cloud.

Here is a list of important locations to keep in mind:
- `/usr/src/project/logs`
  - The logs generated by `boost::log` from the main binary.
- `/usr`

### Setup gcloud Locally
Follow instructions at [gcloud cli](https://cloud.google.com/sdk/docs/install). If you are on Windows you may need to have [putty installed](https://www.putty.org/) for certain SSH operations.

Recommended: for this project is to setup the default project
```sh
gcloud config set project restful-webserver
```
and the compute zone to the one we are using
```sh
gcloud config set compute/zone us-west1-a
```
You may use the following commands to verify that you have things setup correctly
```sh
# check your current active project
gcloud config list project
# check if you are logged in with the corrent user
gcloud auth list
```

Finally with all this setup you can ssh like this
```sh
gcloud compute ssh web-server
```
